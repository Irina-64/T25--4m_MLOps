# Примерные темы для итоговой работы (проекта) курса MLOps
## Рекомендованные направления на основе 14 лабораторных работ

---

## Основные требования для итоговой работы

**Минимальные требования:**
- Полный ML pipeline (ETL → обучение → деплой)
- DVC для версионирования данных
- MLflow для экспериментов и registry
- Тестирование и CI (GitHub Actions)
- Контейнеризация (Docker)
- Документация и README
- Демонстрация (видео или живая презентация)

**Максимальные возможности:**
- Airflow оркестрация
- Feature Store (Feast)
- Kubernetes деплой
- Мониторинг (Prometheus/Grafana)
- Детекция дрейфа
- CI/CD до продакшена

## 20 РЕКОМЕНДОВАННЫХ ТЕМ

### **1. Прогнозирование цен недвижимости (Housing Price Prediction)**

**Описание:**
Система MLOps для предсказания стоимости жилья на основе характеристик (площадь, расположение, возраст и т.д.)

**Ключевые компоненты:**
- Данные: Boston Housing, House Prices Kaggle или собственный датасет
- Модели: Linear Regression, Random Forest, XGBoost
- Pipeline: данные → feature engineering → обучение → регистрация
- API: REST endpoint для получения прогноза цены
- Мониторинг: отслеживание RMSE на реальных данных

**Интеграция ЛР:**
- ЛР1-2: DVC versioning жилых комплексов
- ЛР3: Pipeline для preprocessing (нормализация, кодирование категорий)
- ЛР4-5: MLflow эксперименты и Model Registry
- ЛР6-7: FastAPI + Docker + тесты
- ЛР8: Airflow DAG для переобучения раз в месяц
- ЛР11: Мониторинг ошибок предсказаний

### **2. Классификация изображений (Image Classification - Pets/Flowers)**

**Описание:**
Система для классификации изображений (кошки/собаки или виды цветов) с использованием Transfer Learning

**Ключевые компоненты:**
- Данные: набор изображений (CIFAR, собственный датасет)
- Модели: ResNet50, VGG, EfficientNet (transfer learning)
- Pipeline: загрузка → аугментация → обучение → сохранение весов
- API: endpoint для загрузки изображения и классификации
- Мониторинг: accuracy по классам

**Интеграция ЛР:**
- ЛР1-2: DVC для хранения датасета изображений
- ЛР3: Pipeline обработки (resize, нормализация)
- ЛР4-5: MLflow для логирования accuracy/loss по эпохам
- ЛР6: Docker контейнер с API для инфера
- ЛР7: Unit-тесты для обработки изображений
- ЛР10: K8s для масштабирования инфера
